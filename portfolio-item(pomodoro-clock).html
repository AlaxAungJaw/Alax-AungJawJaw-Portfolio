Pomodoro Clock


by using CSS, JavaScript (Babel) and React.js

"The Pomodoro Clock" with the used of Markups, CSS(flexbox), JavaScript's Loops,
Conditions, Functions with combinations of React's Elements Rendering,
Components & Props, State and Lifecycle, Handling Events.

See the Pen 25 + 5 Clock ("Pomodoro Clock") (freeCodeCamp) by Aung Kyaw Kyaw Alax (@aung-kyaw-kyaw-alax) on CodePen.

  • On the app, by default (on load), displays a value of 5.
  • The length of the session is also display a value of 25 as a default with a string indicating a session is initialized.
  • Either paused or running, the value in the field will always be displayed in mm:ss format(e.g 25:00).
  • When user click "reset", the timer should be stopped, the value within "Break-Length" should return to 5 and the value within "Session-Length" should return to 25, and the left-over time should return to its default state.
  • When user click "-", the value within will decrement by a value of 1, and "+", the value within will increments by a value of 1 in both "Break-Length" & "Session-Length".
  • The app cannot set a session or break length to less than 0 and greater than 60.
  • Once user clicks "Start" and then clicks "Reset" button, the timer begins running from "25:00" as a default that displayed in "Session-Length", regardless the values that have incremented or decremented from the original value of 25.
  • While the timer is running, the remaining should display in mm:ss format (decrementing by a value of 1 and updating the display every 1000ms).
  • Once user clicks "Pause" button, the countdown should pause and when user clicks back "Start" button, the countdown should resume running from the point at which it was paused.
  • Once a session countdown reaches zero (NOTE: timer MUST reach 00:00), then a new countdown begins and a new break countdown will begin from the value currently displayed in the "Break-Length".
  • Also once a break countdown reaches zero (NOTE: timer MUST reach 00:00), a new session countdown also will begin and starts counting down from the value currently displayed in the "Session-Length".
  • When a countdown reaches zero (NOTE: timer MUST reach 00:00), a sound that indicate "time is up" will be played which has been utilized by an HTML5 audio for some short duration.
  • The audio element will stop playing and rewound to the beginning when the element with "Reset" is clicked.
